## 0. 배경 — 왜 스윙인가

- **단타 종목선정 = 데이터로 불가** 확정 (일봉/분봉/60초 전부 변별력 0, 4각도 검증).
  세력 매집이 집계에 묻힘 → 종목선정은 틱으로만(미검증, 수집 중).
- **단타 진입타이밍(거래죽음+평단)** 은 유효하나 종목선정엔 무력.
- **스윙(2주 홀딩) = OHLCV의 본령.** 같은 지표(평단·매물대·MA)를
  "신호 잡기"가 아니라 **"구조"** 로 해석하면 작동.
- 타겟 = 업비트 1개월 상위 폭등군(NEAR +94 / INJ +84 / AKT +57 / IN +56 …).
  "지금 사는 것"이 아니라 "저런 자리를 폭등 前에 고를 수 있는 스캐너".

## 5. ★ 모멘텀 = 강도 (게이트 아님) — 검증됨

자리(지지선근처+추격배제) 통과군 **내부에서** RSI를 강도로 분리:

| 통과군 내부 | 끝수익 | 끝＋ | 손익비 |
|---|---|---|---|
| RSI 중립(45~60) | -1.2% | 41% | 1.41 |
| **RSI 약(<45, 더 눌림)** | **+0.2%** | **51%** | **1.73** |

- RSI **단독**은 약함(과매도 1.10). 근데 **자리 잡은 뒤** 강도로 쓰면 1.51→1.73.
- 끝수익 첫 플러스(+0.2%), 끝＋ 51%.
- **결론: 모멘텀(RSI·OBV·MFI)은 자리 게이트가 아니라, 자리 잡힌 종목의 강도 가점.**

## 6. 확정된 스윙 스캐너 골격

```
[게이트 — 자리(WHERE)]  ← 손익비 1.51
  지지선 근처: MA20 or VWAP or 매물대(POC) 중 ±5% 이내
  AND 추격 배제: VWAP/POC 위 아님 + RSI 60+ 아님 + MA 한참 위 아님

[강도 — 가중(HOW STRONG)]  ← 1.51 → 1.73
  RSI 낮을수록(눌릴수록) 가점
  유동성 스윕 있으면 가점
  고점 많이 빠짐 = MFE 크나 변동성↑ → "고위험고수익" 태그

[단계 표시 — 가르지 않고]
  A단계(과매도·역배열·바닥직후) ↔ B단계(중립·정배열·다진뒤)
  배열·바닥경과로 표시만 (한 구조의 두 시점)

[국면 필터]
  현재 82% 하락장 → BTC.D/시장국면 매크로 게이트 (알파봇 dominance.js 재활용)
```

## 7. 관통 원리

- **살아남은 지표 = 전부 "지지선/세력자리 근처 위치"** (MA·VWAP·매물대·스윕)
- **버려진 지표 = 전부 "방향 모멘텀"** (OBV·MFI·RSI과매도·차트패턴)
- **함정 = 전부 "위/과열/추격"** (VWAP위·POC위·RSI60+·MA위)
- = 단타 상투필터(평단 위 추격금지)와 **동일 원리, 시각만 2주로 확장**.
- 본인 통찰 적중: "OBV는 후행" / "MA를 지지저항으로" / "SMC?" / "해석방식을 바꾸자" / "모멘텀은 자리 보고 강도로".
